# v0.1.0 — 首版发布

**日期**：2026-04-15
**Release**：https://github.com/yosh3289/speedforce/releases/tag/v0.1.0

## 一句话概括

SpeedForce 从零到一，按 M1-M5 五个里程碑一次性交付：托盘应用骨架、核心探测/调度、fyne UI 和 i18n、功能完整版（通知/自启/热重载/日志导出）、CI/CD 和文档。

## 主要内容

### 探测层
- 6 路 HTTPS 探测（3 API + 3 Web 端点）
- 公网 IP + 地理 + ISP 探测（每 5 tick 一次，通过 ipapi.co）
- Anthropic / OpenAI Statuspage JSON API 集成
- 代理感知 HTTP Client（auto/manual/none 三模式）

### 核心调度
- `StateBus` 状态总线 + 非阻塞发布
- `Scheduler` 固定 + 自适应 tick（延迟 Δ>1000ms 或状态翻转进入 10s 快速模式，连续 3 次稳定或 10 分钟上限退出）
- `Notifier` 按服务粒度开关 + 5 分钟抖动冷却
- `ComputeOverall` 整体状态判定（健康/降级/严重）

### UI 层
- getlantern/systray 托盘图标 + 菜单（首版仅左键弹菜单）
- fyne 详情窗口 + 设置窗口（按需创建）
- 三色闪电图标（蓝/黄/红，由用户提供的像素艺术 PNG 用 Python + Pillow 染色生成）
- 中英双语 i18n（`fyne.io/embed` locales）

### 平台相关
- Windows 注册表读代理 + 开机自启 + 命名 mutex 单实例
- darwin/linux 有 build-tag stub（未来填充）

### 开发 & CI
- `--tick` / `--fake-down` 调试 flag
- GitHub Actions CI（windows-latest，test + lint + build）
- Release workflow（tag push 触发）
- 覆盖率：core 89.5%，probe 79.3%

## 构建过程中的经验教训

1. **go.mod 默认版本过高**：`go mod init` 用本地 Go 1.26.2 填充 `go 1.26.2`，但 golangci-lint 构建在 Go 1.25 上会拒绝。降级到 `go 1.25.0`。
2. **Fyne 需要 CGO**：fyne 的 OpenGL 驱动依赖 `go-gl/gl`（CGO 包）。本地需要 MinGW-w64；GitHub `windows-latest` runner 预装了。
3. **Lint timeout**：golangci-lint 加载 fyne 巨大的依赖图需要 >5min，默认 timeout 不够。调到 15min 并 `skip tests`。
4. **PowerShell 参数解析**：`go test -coverprofile=coverage.out` 在 pwsh 里被拆成两个参数（`.out` 当成独立 package）。CI 里的 go 命令都要加 `shell: bash`。
5. **提交邮箱泄露**：默认 git 全局 email 是学校邮箱。被 GitHub "block push exposing private email" 挡。用 `git filter-branch` 把所有历史 commit 的 author/committer email 改成 GitHub noreply 邮箱。
6. **Icon 文件过小**：Pillow `save(format='ICO', append_images=[...])` 方式生成的 ICO 只有 170 字节（第一帧被当主尺寸，append 参数对 ICO 格式无效）。改成单张 256x256 源图 + `sizes=[(16,16)...]` 让 PIL 自动缩放。
